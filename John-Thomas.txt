    1. What is Semantic HTML?

    19. What is the difference between a class and an id? - Give me an example of when I might use each one
IDs have greater specificity than classes. Classes are generic for the sake of multiple uses, whereas IDs are meant for single attributions. If you want all of your images to have a border radius, you would use a class. With an ID, you can refer to it in an href for navigation, such as for a navigation layout near the top which you would want to be able to jump to by clicking on it.

    22. What is a parent child selector? When would this be useful? 
parent > child { }, which allows all directly nested child attributes underneath parent to receive the styling information. This allows you to target upper-level elements in a document without altering those nested within the child. If you've got an article with section (parent) and div (child), any div's that nest inside the original div will not be altered.
